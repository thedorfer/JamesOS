# Security policy

Do not open a public issue for an active vulnerability, credentials, private identifiers, or exploitable deployment details. Use GitHub private vulnerability reporting when available, or contact the repository owner privately.

For architecture and contribution requirements, see [Security model](docs/SECURITY_MODEL.md) and [Contributing](CONTRIBUTING.md).

Include the affected revision, bounded reproduction, impact, and suggested mitigation. Redact secrets and private `JamesOSData`. Do not test against real provider accounts without explicit authorization.
